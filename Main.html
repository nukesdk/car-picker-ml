THE MODEL

How the scoring engine works

Every car is scored on a 0–100 scale across six weighted dimensions. The model balances your hard constraints (budget, seating) with soft preferences (fuel, body type, usage, priority).

WEIGHT: 35%

Budget Fit

Sweet spot is 70–95% of your budget — well-utilized without stretching. Cars below 50% lose points; cars above 100% are filtered out.

WEIGHT: 20%

Body Type Match

Direct match against your selected body types. Multiple selections still rank by how well other attributes align.

WEIGHT: 15%

Fuel Preference

Filters by chosen fuel types. If none picked, scoring falls back to usage suitability (e.g., diesel for highway).

WEIGHT: 10% each

Seating & Usage

Seating must meet your minimum. Usage fine-tunes — city rewards mileage, highway rewards safety & power, off-road rewards ground clearance.

WEIGHT: 10%

Priority Boost

Whatever you picked as your top priority — mileage, safety, features, or performance — gets a 10% boost on its respective attribute.

DEPLOYMENT

Host this on Render in 5 minutes

Since AutoPicker is a fully client-side app (no backend, no database), it deploys as a Static Site. Here's the exact step-by-step.

1
Save the file
Save this entire HTML as index.html in a new folder.
2
Push to GitHub
Create a new repo and upload the file. Or from terminal:
git init
git add .
git commit -m "AutoPicker"
git branch -M main
git remote add origin <your-repo-url>
git push -u origin main
3
Create a Static Site on Render
Go to render.com → New + → Static Site → Connect your GitHub repo.
4
Configure build settings
Use these exact values:
Build Command: (leave empty)
Publish Directory: .
Branch: main
5
Deploy
Click Create Static Site. Render gives you a live URL like https://autopicker.onrender.com in under a minute.
Tip
Add a custom domain from Render's dashboard → Settings → Custom Domains. Free SSL is included automatically.
render.yaml
# Optional: add this file to repo root
# for one-click blueprint deployment

services:
  - type: web
    name: autopicker
    env: static
    buildCommand: ""
    publishDir: .
    pullsBeforeDeploy: true
    headers:
      - path: /*
        name: Cache-Control
        value: no-cache
    routes:
      - type: rewrite
        source: /*
        destination: /index.html
